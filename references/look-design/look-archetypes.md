# Look Archetypes

<!-- SPDX-License-Identifier: CC-BY-4.0 -->

Archetypes are shorthand for a proposed set of signal choices, not universal recipes. Project guidance: convert a look name into explicit, reviewable parameters and references.

Project guidance: a teal-and-orange brief can be translated into a restrained warm-versus-cool separation, then evaluated for skin, neutrals, gamut, and story. The repository does not claim one mandatory teal-and-orange recipe.

Project guidance: a bleach-bypass-inspired treatment can begin with reduced saturation and altered contrast, but its exact relationship to physical processing and its acceptable clipping depend on the chosen reference.

Kodak VISION Color Print Film 2383 / 3383 technical data describes a particular print stock [C: Kodak, "VISION Color Print Film 2383 / 3383 Technical Data"]. Project guidance: a print-emulation transform must be placed in the input domain it expects and evaluated against a chosen reference. It does not guarantee neutral highlights or a theatrical result for arbitrary footage.

Project guidance: a noir brief may call for controlled contrast and localized shaping. Windows, mattes, and tracking are spatial operations outside this v1 LUT-only path [E: repository LIMITS.md].

Project guidance: a pastel brief may use a softer contrast curve and restrained palette. Choose the shadow, highlight, and saturation settings from a reference rather than treating them as mandatory.

Pascarella demonstrates a milky-shadows treatment using lifted shadows and lowered highlights [C: Dante Pascarella, "5 Color Grading Secrets That Make Pro Filmmakers Stand Out"]. It is an example of why shadow-floor targets should be treated as context-dependent appearance choices.

Project guidance: document a look through explicit transforms, parameters, expected input domain, reference frames, and approved waivers.

# Research Method

<!-- SPDX-License-Identifier: CC-BY-4.0 -->

This file defines policy for maintaining the knowledge base. Factual claims must carry an appropriate evidence tag or be rewritten as clearly marked project guidance. Memory is a prompt to research, not a source.

Policy: seek independent corroboration for consequential or changing claims when practical, prioritizing the governing standard, manufacturer document, implementation, or recorded measurement. A single authoritative source may be the correct source for a narrowly specified claim; the citation must state what it supports.

Policy: include an edition, version, or dated identifier when it is available and material. For a mutable web source, record a retrieval date or replace it with a stable named publication or identifier.

Policy: preserve an UNVERIFIED status when the repository cannot establish a source's exact claim. Do not use an opaque redirect as a citation URL. A standard identifier may be cited as an identifier, but the prose must not invent an unverified clause.

Evidence tags identify the kind of support: `[S: ...]` is a standards-body document, `[M: ...]` a manufacturer document, `[E: ...]` a specific repository observation, test, or code artifact, and `[C: ...]` a named craft source. An E tag can support only the identified repository observation; it cannot upgrade an unsupported general claim. Unsupported claims are dropped or marked as project guidance. Stable identifiers are preferred to volatile links.
